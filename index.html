Local LLM Chatbot

A CS1 Assignment for Hands-on AI Exploration

Author: Norman Tiong, Simon Fraser University

Summary Local LLM Chatbot -- students build an interactive chatbot using a real Large Language Model running locally in their browser. They implement conversation memory using lists and dictionaries while discovering AI limitations through direct experimentation. One-click setup via GitHub Codespaces eliminates technical barriers.
Topics
 A practical application of lists, dictionaries, loops, and conditional logic in the context of AI interaction and conversation management.
Audience
 Appropriate for end of term CS1 students with knowledge of basic Python data structures.
Difficulty
 This is an intermediate assignment, taking approximately 2-3 hours for CS1 students to complete all parts.
Strengths
 Students are immediately engaged by building their own AI chatbot that actually works. The assignment naturally leads to discovery moments about AI limitations (like poor math skills) that spark genuine curiosity. The one-click setup eliminates technical barriers, works free at any scale, and students can experiment without cost concerns. Programming concepts are learned in an authentic, motivating context.
Weaknesses
 The underlying LLM behavior can be unpredictable, which may confuse some students initially. The small model sometimes produces inconsistent outputs that require instructor explanation. Students may become more focused on the AI novelty than the programming fundamentals being taught.
Dependencies
 Requires basic Python programming knowledge (variables, functions, input/output). Students need a GitHub account for Codespaces access. No prior AI or machine learning knowledge required. The assignment works entirely in a web browser with no local installation needed.
Variants
 The assignment can be extended to compare different model sizes, experiment with conversation summarization, implement chatbot personalities, or analyze response patterns statistically. Advanced students can explore prompt engineering techniques or build domain-specific chatbots for different applications.

Assignment Structure

Part 1: Basic Chatbot (30 minutes)

Students create a simple chat loop using the provided chat() function:

while True:
    user_input = input("You: ")
    response = chat(user_input)
    print("Bot:", response)

Part 2: Memory System (45 minutes)

Students implement conversation memory using lists and dictionaries, learning how context affects AI responses.

Part 3: Enhanced Features (30 minutes)

Students add conversation history limits and experiment with different AI parameters to observe behavioral changes.

Student Discovery Moments

"I have this code that passes all the tests, but the bot doesn't add numbers correctly. What am I doing wrong?"

This natural discovery leads to discussions about model limitations and the difference between reasoning and pattern matching.

"I thought AI was like a brain... but now I see it's just math and memory."

Students develop more accurate mental models of AI systems through direct interaction.

Technical Implementation

  • Model: Qwen2.5-0.5B (400MB, quantized to 2-bit)
  • Runtime: llama.cpp for efficient CPU inference
  • Platform: GitHub Codespaces (free tier: 2-core, 8GB RAM)
  • Setup time: ~90 seconds from repository fork to working chatbot
  • Languages: Python interface with HTTP API backend

Why This Approach Works

Constraints as Features: The small model's limitations create natural teaching moments. Students encounter context windows, inconsistent outputs, and capability boundaries organically rather than as abstract concepts.

Authentic Experience: Unlike educational simulations, students work with a real AI system. They can experiment with prompt engineering, observe stochastic behavior, and understand why AI applications require careful design.

Scalable and Equitable: Works identically for all students regardless of their hardware or financial resources. Instructors can assign this to 300+ students without budget concerns.

Classroom Tested

Piloted with 16 students in an introductory AI course. Student feedback revealed genuine shifts in understanding about AI systems, moving from "magical thinking" to more technical comprehension of how these systems actually function.

Getting Started

  1. Students click the GitHub Classroom assignment link
  2. Repository auto-forks with personal copy
  3. Click "Open in Codespaces" button
  4. Wait ~90 seconds for environment setup
  5. Run python chatbot.py and start chatting!

Repository: github.com/env3d/cs1-llm-local-chatbot

Extensions and Variations

  • Experiment with conversation summarization techniques

Students leave with both stronger programming fundamentals and practical AI literacy.
